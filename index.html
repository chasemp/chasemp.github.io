IOS granular permissions

Monday, July 22, 2013 » cisco, IOS, security

Preamble

If you have tiered levels of administrators, or you want to
create an account for automation purposes best practice is to
define a custom security level in IOS.

Levels 1 and 15 are defined by default.

Allowing lower levels to see your configuration: IOS

conf t
username backup privilege 3 secret <SECRET>
privilege exec level 3 show startup-config
privilege exec level 3 show
wr mem

Allowing lower levels to see your configuration: FreeRADIUS:

/etc/freeradius/users


backup
    Service-Type = NAS-Prompt-User,
    cisco-avpair = "shell:priv-lvl=3",
    Auth-Type = System,

Looks Like


cisco#show start
Building configuration...

--CONFIGURATION--
end

Defining the next user tier for NAT information: IOS

conf t
username natinfo privilege 4 secret <SECRET>
privilege exec level 4 show nat64 statistics
wr mem

Defining the next user tier for NAT information: FreeRADIUS


natinfo
    Service-Type = NAS-Prompt-User,
    cisco-avpair = "shell:priv-lvl=4",
    Auth-Type = System,

Looks Like

cisco#show ip nat statistics
--STATS--

NOTES

  • Permissions stack up which means a user at Level 4 can also issue 'show startup-configuration'
  • 'show running-config' command does not work in this way for the IOS devices I have tried.

This does not work how you think it does:

privilege exec level 3 show running-config

Preamble

Some commands do not support a proxy (RE: telnet). Other
times it is just easier to do a one-off instance rather
than mess with environment settings. A lot of people use
this with Tor.

ProxyChains

Allows you to send traffic through a proxy without the command
itself having foreknowledge.

aptitude install proxychains

Config files

Pick one:

  • ./proxychains.conf
  • $(HOME)/.proxychains/proxychains.conf
  • /etc/proxychains.conf

/etc/proxychains.conf the global option


strict_chain
quiet_mode
tcp_read_time_out 15000
tcp_connect_time_out 8000
[ProxyList]
#replace this with your proxy
http    192.168.1.1 8080

allowing env magic via sudo

Thursday, July 18, 2013 » sudo

Preamble

In an environment with heavy automation and numerous service account
users it is easy to end up in a situation where compromising one service
becomes compromising an entire system.

Granting permission to a single command

vi /etc/sudoers.d/makingpie

myuser ALL = NOPASSWD: /usr/local/bin/makepie

Looks Like

myuser:#sudo makepie
>>>>mmmmm...pie

What if my user needs to call out to a proxy but 'makepie' doesn't support it?

We use proxychains.

sudo LD_PRELOAD=libproxychains.so.3 makepie

Sudo is extra granular and doesn't like this idea.

sudo: sorry, you are not allowed to set the following environment variables: LD_PRELOAD

The heavy handed way

Don't reset user enviroments when using sudo

visudo

Replace: “Defaults env_reset” with: “Defaults !env_reset”

Piecemeal way

vi /etc/sudoers.d/ld_preload

#allow users to use proxychains env variable (using sudo)
Defaults env_keep += "LD_PRELOAD"

making trivial debian packages

Wednesday, July 17, 2013 » deb, debian, dpkg

Preamble

When possible it is beneficial having a canonical reference for resources installed
on a server. Binary packages have built in logic for upgrade, downgrade,
install, etc. This makes them an attractive, yet sometimes, overcomplicated
distribution mechanisms.

I am going to demonstrate making a simple script into a debian binary
and installing it. Adding this to an in-house repo would be a nice next
step.

Getting Packages

Assuming you have dpkg but if not we need it. Fakeroot we use to make packages
with the right permissions on files.

aptitude install dpkg fakeroot lintian

We will be using dpkg-deb which is generally considered 'built in' to debian.


me@vm:# dpkg -S /usr/bin/dpkg-deb
dpkg: /usr/bin/dpkg-debø

Package: my-files.

Starting at version 1.0 and meant for all architectures. 'all' is a common
assignment for architecture when the files/scripts are interpreted such as
with python or ruby.

mkdir my-files_1.0_all

Making my package tree

mkdir -p my-files_1.0_all/usr/local/bin/
mkdir -p my-files_1.0_all/DEBIAN

My script: aptitude easter eggs

nano my-files_1.0_all/usr/local/bin/moo


#!/bin/bash
aptitude -v -v -v -v -v moo

Adding my 'control' file

#this file contains my package metadata
nano my-files_1.0_all/DEBIAN/control

Looks Like

Priority: optional

Maintainer: Chase 
Architecture: all
Version: 1.0
Description: stuff I need

Making my script executable on install

#postint will be run at the tail end of install by default
echo "chmod +x /usr/local/bin/moo" > my-files_1.0_all/DEBIAN/postinst
#set correct permissions to avoid build errors
chmod 555 my-files_1.0_all/DEBIAN/postinst

My full package tree

|   |-- control
|   `-- postinst

Building my binary

Fakeroot should fix up your permissions, permissions are taken very literally by
dpkg-deb. So if you build with a user that does not exist on a host that installs
this package things break. But even if you don't run into that it is considered
bad form to leave your user permissions as artifacts on binary packaged files.

fakeroot dpkg-deb -b my-files_1.0_all/

simple nat outbound asa

Monday, July 15, 2013 » asa, cisco, nat

Preamble

NAT on the ASA can get complicated but 90% of remote sites need
some version of this basic configuration.

Interfaces

Establish your NAT roles.


!
interface Ethernet0/0
 nameif Outside
 security-level 0
 ip address  
!
interface Ethernet0/1
 nameif Inside
 security-level 100
 ip address

Client Pool

Define who gets an address for going out.


object network NAT-CLIENTS
 !can be a subnet also, etc
 range 192.168.1.1 192.168.1.10

NAT Enable

Associate your clients with your external interface for NAT.


nat (Inside,Outside) source dynamic NAT-CLIENTS interface

NOTES

  • This does not require extra global IP's.
  • This dynamic IP may supercede intended static NAT IP's
  • Firewall restrictions still apply. See docs on security levels.

    #
    # This plugin supports version 1.1 or 1.1.6 and newer of the Stomp rubygem
    # the versions between those had multi threading issues.
    #
    # For all versions you can configure it as follows:

    #    connector = stomp

    #    plugin.stomp.port = 6163

    #    plugin.stomp.password = secret

      # Receives a message from the Stomp connection
      def receive
        Log.debug("Waiting for a message from Stomp")

        # When the Stomp library > 1.2.0 is mid reconnecting due to its reliable connection
        # handling it sets the connection to closed.  If we happen to be receiving at just
        # that time we will get an exception warning about the closed connection so handling
        # that here with a sleep and a retry.
        begin
          msg = @connection.receive
        rescue ::Stomp::Error::NoCurrentConnection
          sleep 1
          retry
        end

        Message.new(msg.body, msg, :base64 => @base64, :headers => msg.headers)
      end

      # Sends a message to the Stomp connection
      def publish(msg)
        msg.base64_encode! if @base64

        raise "Cannot set specific reply to targets with the STOMP plugin" if msg.reply_to

        if msg.type == :direct_request
          msg.discovered_hosts.each do |node|
            target = make_target(msg.agent, msg.type, msg.collective, node)

            Log.debug("Sending a direct message to STOMP target '#{target}'")

            publish_msg(target, msg.payload)
          end
        else
          target = make_target(msg.agent, msg.type, msg.collective)

          Log.debug("Sending a broadcast message to STOMP target '#{target}'")

          publish_msg(target, msg.payload)
        end
      end

      # Subscribe to a topic or queue
      def subscribe(agent, type, collective)
        source = make_target(agent, type, collective)

        unless @subscriptions.include?(source)
          Log.debug("Subscribing to #{source}")
          @connection.subscribe(source)
          @subscriptions << source
        end
      rescue ::Stomp::Error::DuplicateSubscription
        Log.debug("Received subscription for #{source[:name]} but already had a subscription, ignoring")
      end

      # Actually sends the message to the middleware
      def publish_msg(target, msg)
        # deal with deprecation warnings in newer stomp gems
        if @connection.respond_to?("publish")
          @connection.publish(target, msg, msgheaders)
        else
          @connection.send(target, msg, msgheaders)
        end
      end

      # Subscribe to a topic or queue
      def unsubscribe(agent, type, collective)
        source = make_target(agent, type, collective)

        Log.debug("Unsubscribing from #{source}")
        @connection.unsubscribe(source)
        @subscriptions.delete(source)
      end

      # Disconnects from the Stomp connection
      def disconnect
        Log.debug("Disconnecting from Stomp")
        @connection.disconnect
      end

      def msgheaders
        headers = {}
        headers = {"priority" => @msgpriority} if @msgpriority > 0

        return headers
      end

      # looks in the environment first then in the config file
      # for a specific option, accepts an optional default.
      #
      # raises an exception when it cant find a value anywhere
      def get_env_or_option(env, opt, default=nil)
        return ENV[env] if ENV.include?(env)
        return @config.pluginconf[opt] if @config.pluginconf.include?(opt)
        return default if default

        raise("No #{env} environment or plugin.#{opt} configuration option given")
      end

      # looks for a config option, accepts an optional default
      #
      # raises an exception when it cant find a value anywhere
      def get_option(opt, default=nil)
        return @config.pluginconf[opt] if @config.pluginconf.include?(opt)
        return default if default

        raise("No plugin.#{opt} configuration option given")
      end

      # gets a boolean option from the config, supports y/n/true/false/1/0
      def get_bool_option(opt, default)
        return default unless @config.pluginconf.include?(opt)

        val = @config.pluginconf[opt]

        if val =~ /^1|yes|true/
          return true
        elsif val =~ /^0|no|false/
          return false
        else
          return default
        end
      end

      def make_target(agent, type, collective, target_node=nil)
        raise("Unknown target type #{type}") unless [:directed, :broadcast, :reply, :request, :direct_request].include?(type)
        raise("Unknown collective '#{collective}' known collectives are '#{@config.collectives.join ', '}'") unless @config.collectives.include?(collective)

        prefix = @config.topicprefix

        case type
          when :reply
            suffix = :reply
          when :broadcast
            suffix = :command
          when :request
            suffix = :command
          when :direct_request
            agent = nil
            prefix = @config.queueprefix
            suffix = Digest::MD5.hexdigest(target_node)
          when :directed
            agent = nil
            prefix = @config.queueprefix
            # use a md5 since hostnames might have illegal characters that
            # the middleware dont understand
            suffix = Digest::MD5.hexdigest(@config.identity)
        end

        ["#{prefix}#{collective}", agent, suffix].compact.join(@config.topicsep)
      end
    end

Preamble

Updating only relevant Puppet configurations sections

As a puppet installation grows the list of used modules can get large. Especially,
if you are pulling from https://forge.puppetlabs.com/. Testing in its final stages
is usually done via puppet to ensure repeatability. The test runs get longer and longer.

A full catalog run with no expected changes



sudo /usr/bin/puppet agent --test --logdest syslog





Info: Retrieving plugin

Info: Caching catalog for 

Info: Applying configuration version '1371116293'

Finished catalog run in 16.35 seconds

If you need to rerun this every time you are tweaking a config file 16 seconds can be a long time.

If you are only updating a module called 'dhcp' you can do:



sudo /usr/bin/puppet agent --test --logdest syslog --tags dhcp





Info: Retrieving plugin

Info: Caching catalog for 

Info: Applying configuration version '1371116293'

Finished catalog run in 8.46 seconds

This cut the time in half. Doing tagged runs still compiles the entire update. That is to
say, Puppet still runs in the usual way figuring out dependencies and such, but when it
comes to applicable changes it only looks for changes at the specified tagged resources.

If you put '--debug' into the update command you will see a lot of:

Debug: /Stage[main]/Mcollective::Client/Package[mcollective-client]: Not tagged with dhcp

This resource does not match the tag specified, and as such will not be updated.

Tags can be explicit or implicit. That is to say you can assign them and there are some
that are assigned by design. All class/module names are also tags.

This automatically creates a tag called 'hiera':



class hiera {

}

Resource types are generally given a tag as well.

To update file resources only:



sudo /usr/bin/puppet agent --test --logdest syslog --tags file

code test beginnings

Friday, June 07, 2013 » python, test

Preamble

Testing your code as you go. Writing 'production' code is really a set of practices
rather than a specific coding ability. Everyone breaks things. When something breaks
we will all fix it. The trick is to always know when something breaks.

How I currently see the world:

  • Write code
  • Run unit tests

It continues on ...

  • Run integration tests
  • Run QA/Functionality testing
  • Test at scale
  • Test monitoring
  • Test Security

There are a lot of ways to break down this basic idea. Some much better than this. But I
feel like if you can visualize these 6 you are going in the right direction.

Write Code

def what():
    return '123'

if __name__ == '__main__':
    print what()

This is my big application. When I run it I see.


   123

Unit Testing - Testing Small Things

My idea of unittesting is if every function and class takes care of itself the entire
codebase is better off.

Doctest is a way to write small tests into the docstring. Some people do not consider this
to be valid unittesting. I think it's a good start.

def what():
    """
    >>> what()
    '123'
    """
    return '123'

if __name__ == '__main__':
    what()

'python -m doctest t.py' returns nothing.

But:

'python -m doctest t.py  -v' returns useful output:


Trying:
    what()
Expecting:
    '123'
ok
1 items had no tests:
    t
1items passed all tests:
   1 tests in t.what
1 tests in 2 items.
1 passed and 0 failed.
Test passed.

Everything passed. In the most basic sense the code in the doctstring returned expected results.

What if my function code changes and the doctest becomes invalid?

Change it to:

def what():
    """
    >>> what()
    '1234'
    """
    return '123'

Now we see:


:~$ python -m doctest t.py -v
Trying:
    what()

Where does SNMP look for MIBs?

net-snmp-config --default-mibdirs

Make user specific directory (needs to be in above path)

mkdir -p /home/<me>/.snmp/mibs

Enable MIB

net-snmp-config --snmpconfpath

Edit your user snmp configuration (if path is included in above command)

n /home/<me>/.snmp/snmp.conf
mib +CISCO-UNIFIED-FIREWALL-MIB

Try walking the top OID tree

simple iptables uses

Tuesday, June 04, 2013 » iptables

Allow ssh from only 192 network (presumably your internal)

iptables -A INPUT -i eth0 -p tcp -s 192.168.0.0/16 --dport 22 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A OUTPUT -o eth0 -p tcp --sport 22 -m state --state ESTABLISHED -j ACCEPT

Allow SCCP (VOIP Protocol)

iptables -A INPUT -i eth0 -p tcp --dport 1720 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A OUTPUT -o eth0 -p tcp --sport 1720 -m state --state ESTABLISHED -j ACCEPT

Allow h.323 (VOIP)

iptables -A INPUT -i eth0 -p tcp --dport 2000 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A OUTPUT -o eth0 -p tcp --sport 2000 -m state --state ESTABLISHED -j ACCEPT

Allow ICMP

iptables -A OUTPUT -p icmp --icmp-type echo-request -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-reply -j ACCEPT

Allow DNS queries, etc

iptables -A OUTPUT -p udp -o eth0 --dport 53 -j ACCEPT
iptables -A INPUT -p udp -i eth0 --sport 53 -j ACCEPT

Allow mail sending

iptables -A INPUT -i eth0 -p tcp --dport 25 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A OUTPUT -o eth0 -p tcp --sport 25 -m state --state ESTABLISHED -j ACCEPT

Allowing SSH and HTTP and self sourced traffic

iptables -A INPUT -i eth0 -p tcp --dport 22 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A OUTPUT -o eth0 -p tcp --sport 22 -m state --state ESTABLISHED -j ACCEPT

query puppetdb facts from cli

Monday, June 03, 2013 » facter, puppet

Preamble

A need to get host facts when they are not up. PuppetDB has cached them, and
I like the CLI.

This doesn't work like you think it does

puppet facts find <myhost.com>

This always returns the facts for the puppetmaster itself.

Setting Terminus

If you want to query the puppetdb store directly set the terminus.

puppet facts find <myhost.com> --terminus puppetdb

This returns the 'right stuff'. Beware it could be old because it's last cached values.

Readability Counts

puppet facts find <myhost.com> --terminus puppetdb | python -m json.tool
